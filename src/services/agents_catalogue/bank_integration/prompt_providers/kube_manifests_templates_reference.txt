## Template Structure

Each deployment file contains THREE deployments:
1. **Main Deployment** - Standard worker
2. **Canary Deployment** - Canary testing (conditional)
3. **Baseline Deployment** - Baseline testing (conditional)

## Core Template Pattern

Based on IDFC reference templates, here's the complete template structure:

### For Standard Operations (Core/UPI/IMPS/NEFT/RTGS):

**Filename**: `fts-live-worker-newbank-{operation}-deployment.yaml`
**Worker Args**: `newbank::{operation}`

Example for UPI Initiate Transfer:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  labels:
    bu: razorpayx
    name: {{ .Values.name }}-live-worker-newbank-upi-initiate-transfer
  name: {{ .Values.name }}-live-worker-newbank-upi-initiate-transfer
  namespace: '{{ .Values.namespace }}'
spec:
  progressDeadlineSeconds: 600
  replicas: {{.Values.fts_live_worker_newbank_upi_initiate_transfer_replicas}}
  revisionHistoryLimit: 10
  selector:
    matchLabels:
      name: {{ .Values.name }}-live-worker-newbank-upi-initiate-transfer
  strategy:
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%
    type: RollingUpdate
  template:
    metadata:
      annotations:
        prometheus.app/port: "8081"
        prometheus.app/scrape: "true"
      creationTimestamp: null
      labels:
        bu: razorpayx
        name: {{ .Values.name }}-live-worker-newbank-upi-initiate-transfer
      name: {{ .Values.name }}-live-worker-newbank-upi-initiate-transfer
    spec:
      serviceAccountName: {{ .Values.service_account_name }}
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
            - podAffinityTerm:
                labelSelector:
                  matchExpressions:
                    - key: name
                      operator: In
                      values:
                        - {{ .Values.name }}-live-worker-newbank-upi-initiate-transfer
                topologyKey: kubernetes.io/hostname
              weight: 100
      automountServiceAccountToken: true
      containers:
        - args:
            - worker
            - newbank::upi::initiate_transfer
          env:
            - name: APP_MODE
              value: live
            - name: APP_ENV
              value: {{ .Values.app_env }}
            - name: JAEGER_HOSTNAME
              valueFrom:
                fieldRef:
                  apiVersion: v1
                  fieldPath: spec.nodeName
          envFrom:
            - secretRef:
                name: {{ .Values.name }}-live
                optional: false
          image: c.rzp.io/razorpay/{{ .Values.fts_image }}
          imagePullPolicy: IfNotPresent
          livenessProbe:
            failureThreshold: 3
            httpGet:
              path: /status
              port: 80
              scheme: HTTP
            initialDelaySeconds: 10
            periodSeconds: 10
            successThreshold: 1
            timeoutSeconds: 2
          name: live
          readinessProbe:
            failureThreshold: 3
            httpGet:
              path: /status
              port: 80
              scheme: HTTP
            initialDelaySeconds: 10
            periodSeconds: 5
            successThreshold: 1
            timeoutSeconds: 2
          resources:
            requests:
              cpu: {{.Values.fts_worker_cpu_requsts}}
              memory: {{.Values.fts_worker_memory_requsts}}
            limits:
              memory: {{.Values.fts_worker_memory_limits}}
          terminationMessagePath: /dev/termination-log
          terminationMessagePolicy: File
          volumeMounts:
            - mountPath: /config/
              name: fts-app-config
              readOnly: true
      dnsPolicy: ClusterFirst
      dnsConfig:
        options:
          - name: ndots
            value: "1"
      nodeSelector:
        {{ .Values.queues_node_selector }}: ""
      restartPolicy: Always
      schedulerName: default-scheduler
      securityContext: {}
      terminationGracePeriodSeconds: 60
      volumes:
        - configMap:
            defaultMode: 420
            name: fts-app-config
          name: fts-app-config

{{ if .Values.canary_baseline_enable }}
---
# CANARY DEPLOYMENT (similar structure with -canary suffix and INSTANCE_TYPE: canary)
# BASELINE DEPLOYMENT (similar structure with -baseline suffix and INSTANCE_TYPE: baseline)
{{ end }}
```

## Template Replacement Rules

When generating templates for a new bank, replace:
1. **Bank Name**: `idfc` → `newbank` (actual bank name in lowercase)
2. **Variable Names**: `fts_live_worker_idfc_*` → `fts_live_worker_newbank_*`
3. **Worker Args**: `idfc::*` → `newbank::*`
4. **Labels/Names**: All references to bank name in metadata and selectors

## Canary/Baseline Additions

Each template file includes conditional canary and baseline deployments:
- **Canary**: Adds `INSTANCE_TYPE: canary` and `CONCURRENCY: '1'` env vars
- **Baseline**: Adds `INSTANCE_TYPE: baseline` and `CONCURRENCY: '1'` env vars
- **Resource Limits**: Uses `fts_canary_baseline_cpu_requests` and `fts_canary_baseline_memory_requests`
- **Config Maps**: Canary uses `fts-app-config-canary`, baseline uses `fts-app-config`

## File Generation Priority

1. **Core Operations** (initiate_transfer, check_transfer_status) - Essential
2. **UPI Operations** - Most common payment method
3. **IMPS Operations** - Immediate payment service
4. **NEFT Operations** - National electronic funds transfer
5. **RTGS Operations** - Real-time gross settlement
6. **Direct Operations** - Bank-specific direct API integrations

## Key Points:
- All deployments are conditional on `{{ if .Values.canary_baseline_enable }}`
- Each file contains 1 main + 2 conditional deployments (canary/baseline)
- Worker arguments determine the queue and processing logic
- Resource requests/limits are templated for flexibility
- Prometheus annotations are included for monitoring
- Anti-affinity rules prevent multiple pods on same node

## Template Replacement:
- `newbank` → actual bank name (lowercase)
- Example: For Bank of Maharashtra → `bom`
