We need to implement a shadow mode for our payment processing system to validate a new payout service before fully switching over to it. The current payout system has been reliable but lacks modern features and scalability needed for our growing user base.

The shadow mode should:
1. Process all payout requests through both the existing system and the new system
2. Compare the results from both systems for accuracy and performance
3. Log any discrepancies for analysis
4. Not affect the customer experience - customers should only see results from the current system
5. Collect metrics on processing time, success rates, and error types
6. Be easily toggled on/off without service disruption

This shadow mode will help us validate the new system's reliability and performance in a production environment before fully migrating to it.
