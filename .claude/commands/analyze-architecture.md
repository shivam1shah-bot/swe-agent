# Analyze Architecture Command

## Purpose
Analyze the current codebase architecture and provide insights on structure, patterns, and potential improvements.

## Instructions
When this command is invoked, perform the following:

1. **Review Project Structure**
   - Examine the layered architecture (API, Service, Repository, Model, Provider)
   - Identify key components and their relationships
   - Check for adherence to clean architecture principles

2. **Analyze Patterns**
   - Review use of design patterns (repository, singleton, factory, etc.)
   - Check dependency injection and inversion of control
   - Examine error handling and validation patterns

3. **Assess Data Flow**
   - Trace request flow through layers
   - Review task execution pipeline
   - Analyze integration points and external dependencies

4. **Provide Report**
   - Summarize current architecture state
   - Highlight strengths and best practices
   - Identify potential improvements or concerns
   - Suggest refactoring opportunities if needed

## Output Format
Use the "architectural-report" output style if available.
